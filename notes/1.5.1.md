# ModbusLink v1.5.1 版本发布说明

## 核心更新：TCP 传输层连接超时分离

### 问题背景

在工业现场环境中，网络连接建立和数据处理往往需要不同的超时策略：

1. **连接建立**：通常期望快速失败，避免长时间等待
2. **数据操作**：可能需要较长的等待时间，特别是慢速设备或高延迟网络

在 v1.5.1 之前，TCP 传输层只有一个统一的 `timeout` 参数，无法区分这两种场景。

### 解决方案

`1.5.1` 引入了细粒度的超时控制，为 `SyncTcpTransport` 和 `AsyncTcpTransport` 添加了独立的 `connection_timeout` 参数。

## ✨ 新特性

### 1. 独立的连接超时参数

新增 `connection_timeout` 参数，专门用于控制 TCP 连接建立的超时时间：

```python
# 同步版本
transport = SyncTcpTransport(
    host="192.168.1.100",
    port=502,
    timeout=10.0,              # 数据操作超时（发送/接收）
    connection_timeout=3.0      # 连接建立超时
)

# 异步版本
transport = AsyncTcpTransport(
    host="192.168.1.100",
    port=502,
    timeout=10.0,              # 数据操作超时（发送/接收）
    connection_timeout=3.0      # 连接建立超时
)
```

### 2. 向后兼容设计

- `connection_timeout` 是可选参数，默认为 `None`
- 当 `connection_timeout=None` 时，自动使用 `timeout` 的值
- 现有代码无需任何修改即可正常工作

### 3. 应用场景优化

#### 场景 1：快速失败连接，慢速数据处理

```python
# 适用于网络不稳定但设备响应慢的环境
transport = SyncTcpTransport(
    host="192.168.1.100",
    port=502,
    timeout=10.0,              # 等待设备响应 10 秒
    connection_timeout=2.0      # 连接建立只等 2 秒
)
```

#### 场景 2：高延迟网络环境

```python
# 适用于跨网段或 DTU 通信
transport = AsyncTcpTransport(
    host="remote-device.example.com",
    port=502,
    timeout=30.0,              # 慢速网络，数据操作等 30 秒
    connection_timeout=10.0    # 连接建立等 10 秒
)
```

## 🛠️ 变更详情

### Transport 层

- **SyncTcpTransport**:
  - 新增参数 `connection_timeout: Optional[float] = None`
  - `open()` 方法使用 `connection_timeout` 替代 `timeout` 进行连接建立
  - 更新 `__repr__` 方法显示 `connection_timeout`

- **AsyncTcpTransport**:
  - 新增参数 `connection_timeout: Optional[float] = None`
  - `open()` 方法使用 `connection_timeout` 替代 `timeout` 进行连接建立
  - 更新 `__repr__` 方法显示 `connection_timeout`

### 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `timeout` | float | 1.0 | 操作超时时间，用于数据发送和接收 |
| `connection_timeout` | Optional[float] | None | 连接建立超时时间，默认等于 timeout |

## ⚠️ 兼容性说明

- **完全向后兼容**：现有代码无需任何修改
- **无破坏性变更**：所有现有功能保持不变
- **Python 版本**：Python >= 3.9

## 📈 升级建议

建议在以下场景使用新的 `connection_timeout` 参数：

1. **网络环境不稳定**：快速失败连接，避免长时间等待
2. **设备响应慢**：允许较长的数据处理时间
3. **高延迟网络**：跨网段或 DTU 通信环境
4. **生产环境优化**：根据实际网络状况调优超时参数

## 使用示例

### 基本用法

```python
from modbuslink import SyncModbusClient, SyncTcpTransport

# 使用默认值（向后兼容）
transport1 = SyncTcpTransport(host="192.168.1.100", port=502, timeout=10.0)
# connection_timeout 默认为 10.0

# 设置独立的连接超时
transport2 = SyncTcpTransport(
    host="192.168.1.100",
    port=502,
    timeout=10.0,           # 操作超时 10 秒
    connection_timeout=3.0    # 连接超时 3 秒
)

client = SyncModbusClient(transport2)
with client:
    data = client.read_holding_registers(slave_id=1, start_address=0, quantity=10)
```

### 异步版本

```python
import asyncio
from modbuslink import AsyncModbusClient, AsyncTcpTransport

async def main():
    transport = AsyncTcpTransport(
        host="192.168.1.100",
        port=502,
        timeout=10.0,
        connection_timeout=3.0
    )
    
    async with AsyncModbusClient(transport) as client:
        data = await client.read_holding_registers(slave_id=1, start_address=0, quantity=10)
        print(f"数据: {data}")

asyncio.run(main())
```
