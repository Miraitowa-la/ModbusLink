# ModbusLink v1.3.2 版本发布说明

## 新功能：支持统一语言配置系统

### 全局语言设置

现在可以通过统一的语言配置系统来设置整个库的语言，包括日志和异常消息。新的 `set_language()` 函数允许全局设置语言，影响所有模块：

```Python
from modbuslink import Language, set_language

# 设置为英文
set_language(Language.EN)

# 设置为中文（默认）
set_language(Language.CN)
```

### 更新的日志配置

`ModbusLogger.setup_logging()` 函数不再需要单独的语言参数，因为语言设置现在是全局的（当然为兼容以前，依旧可在该函数设置语言属性，全局的）：

```Python
from modbuslink import Language, set_language, ModbusLogger
import logging

# 设置全局语言（影响日志和异常消息）
set_language(Language.EN)

# 配置日志
ModbusLogger.setup_logging(
    level=logging.DEBUG,
    enable_debug=True,
    log_file='modbus_debug.log'
    # language=Language.EN # 该属性仍然保留
)
```

## API 新增内容

### 新增模块

`language.py` (src\modbuslink\common\language.py)：新增统一语言配置模块
- Language.CN / Language.EN - 语言常量
- set_language(lang) - 全局语言设置函数
- get_language() - 获取当前语言设置
- get_message(cn, en) - 根据当前语言获取消息

### 更新的异常处理

`exceptions.py` (src\modbuslink\common\exceptions.py)：异常类现在支持双语消息
- 所有异常类（ConnectionError, TimeoutError, CRCError, InvalidResponseError, ModbusException）都支持 cn 和 en 参数
- ModbusException 现在根据全局语言设置显示适当的消息

### 更新的日志系统

`logging.py` (src\modbuslink\utils\logging.py)：更新为使用共享语言配置
- BilingualLogger 类现在使用统一的语言设置
- 移除了本地语言配置，使用来自 common/language.py 的全局设置

### 多个文件中英文适配

修改了以下文件的中英文适配（当然可能还有我没看见的地方）：
- src\modbuslink\client
- src\modbuslink\common
- src\modbuslink\server
- src\modbuslink\transport
- src\modbuslink\utils

### 中英文示例

示例目录结构已更新，从原有的中英文混合示例改为分离的目录结构：
- examples\cn - 中文示例
- examples\en - 英文示例
