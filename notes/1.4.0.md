# ModbusLink v1.4.0 版本发布说明

## 1. 概述 (Executive Summary)

本次重构旨在解决项目架构中的冗余问题，统一命名规范，并提升核心组件的性能与稳定性。主要变更点如下：

1. **架构合并**：不再将同步（Sync）与异步（Async）的传输层实现分离在不同文件中，而是按协议类型合并。
2. **命名规范化**：
    - **类名**：同步类统一增加 `Sync` 前缀（如 `ModbusClient` -> `SyncModbusClient`）。
    - **文件名**：移除服务器层冗余的 `async_` 前缀；传输层增加 `_transport` 后缀。
    - **异常名**：重命名异常以符合 PEP 8 驼峰命名法，并建立清晰的继承树。
3. **模块重组**：日志模块 (`logging`) 从工具层 (`utils`) 提升至通用层 (`common`)。
4. **性能与功能**：CRC 校验引入查表法，新增 LRC 校验，语言模块支持上下文隔离（ContextVars）。

## 3. 核心 API 变更速查 (Breaking Changes & Migration)

如果您直接从顶层包 `modbuslink` 导入对象，请参考下表进行修改。

### 3.1 客户端与传输层 (Client & Transport)

| **对象类型**     | **原名称/路径**        | **新名称/路径**           | **说明**                                                |
|--------------|-------------------|----------------------|-------------------------------------------------------|
| **同步客户端**    | `ModbusClient`    | `SyncModbusClient`   | 增加前缀以明确同步特性                                           |
| **同步 TCP**   | `TcpTransport`    | `SyncTcpTransport`   | 增加前缀，所在文件变为 `tcp_transport.py`                        |
| **同步 RTU**   | `RtuTransport`    | `SyncRtuTransport`   | 增加前缀，所在文件变为 `rtu_transport.py`                        |
| **同步 ASCII** | `AsciiTransport`  | `SyncAsciiTransport` | 增加前缀，所在文件变为 `ascii_transport.py`                      |
| **异步传输**     | `Async*Transport` | `Async*Transport`    | 类名不变，但所在文件已合并（如 `async_tcp.py` -> `tcp_transport.py`） |

### 3.2 异常类 (Exceptions)

异常模块 `modbuslink.common.exceptions` 进行了全面重写。旧的异常类名大部分已被废弃。

| **原异常类**               | **新异常类**             | **说明**                                   |
|------------------------|----------------------|------------------------------------------|
| `ConnectionError`      | `ConnectError`       | 避免与 Python 内置 `ConnectionError` 混淆       |
| `TimeoutError`         | `TimeOutError`       | 避免与 Python 内置 `TimeoutError` 混淆          |
| `CRCError`             | `CrcError`           | 修正为驼峰命名法                                 |
| `InvalidResponseError` | `InvalidReplyError`  | 语义修正                                     |
| (无)                    | `LrcError`           | [新增] ASCII 模式校验错误                        |
| (无)                    | `CommunicationError` | [新增] `ConnectError` 和 `TimeOutError` 的父类 |
| (无)                    | `ValidationError`    | [新增] `CrcError` 等校验错误的父类                 |

### 3.3 其他移除项 (Removals)

- `RS485Settings`: `1.4.0` 版本目前移除了对 `RS485` 的支持，后续会重新添加。
- `utils.logging`: 该路径下的模块已被移除，请修改为 common.logging。

## 4.模块详细变更说明 (Detailed Changes)

### 4.1 🛠️ Common (通用组件)

- **Logging (`common/logging.py`)**:
    - **位置变更**: 从 `utils` 移动到 `common`。
    - **新增**: `BilingualLogger` 类及其导出。
    - **API**: `get_logger` 现在返回 `BilingualLogger` 实例，支持中英双语日志记录。
- **Language (`common/language.py`)**:
    - **线程安全**: 内部从全局变量改为 `contextvars.ContextVar`，完美支持异步并发和多线程环境下的多语言切换。
    - **新增**: `use_language(lang)` 上下文管理器 (`with use_language(Language.EN): ...`)。

### 4.2 🚚 Transport (传输层)

- 文件合并: 所有的 async_*.py 传输文件都被合并到对应协议的主文件中。
- API 增强:
    - `send_and_receive` 方法新增 `timeout` 参数，允许对单次请求指定超时时间。
    - 同步传输类现明确使用 `threading.Lock`。
    - 异步传输类现明确使用 `asyncio.Lock`。

### 4.3 🖥️ Server (服务器层)

- **文件重命名**:
  `async_tcp_server.py` -> `tcp_server.py`
  `async_rtu_server.py` -> `rtu_server.py`
  `async_ascii_server.py` -> `ascii_server.py`
- **架构优化**:
  引入 `AsyncSerialModbusServer` (`serial_server.py`) 作为 RTU 和 ASCII 服务器的基类，统一管理串口连接逻辑。
- **DataStore**:
  新增 `add_callback(area, callback)` 方法，支持监听数据区的写入变化（观察者模式）。

### 4.4 🔧 Utils (工具层)

- **CRC (`crc.py`)**:
    - 重构为**查表法**实现，计算速度显著提升。
    - 保留旧算法作为备选 (`use_table=False`)。
- **Coder (`coder.py`)**:
    - 重写 `PayloadCoder` 内部逻辑，统一大小端和高低字处理，修复潜在转换 Bug。
    - 字符串处理由 `.rstrip` 改为 C 风格截断（遇到 `\x00` 停止）。
- **LRC (`lrc.py`)**:
    - 新增模块，提供 ASCII 模式所需的 LRC 校验算法。

## 代码迁移示例 (Code Migration Examples)

### 示例 1：同步 TCP 客户端 (Sync TCP Client)

```python
# ================= OLD =================
from modbuslink import ModbusClient, TcpTransport
from modbuslink import ConnectionError, TimeoutError

try:
    # 建立传输层
    transport = TcpTransport(host="127.0.0.1")
    # 建立客户端
    client = ModbusClient(transport)

    data = client.read_holding_registers(0, 10)

except (ConnectionError, TimeoutError) as e:
    print(f"Error: {e}")

# ================= NEW =================
# 1. 类名变更: ModbusClient -> SyncModbusClient
# 2. 类名变更: TcpTransport -> SyncTcpTransport
from modbuslink import SyncModbusClient, SyncTcpTransport
# 3. 异常类变更
from modbuslink import ConnectError, TimeOutError

try:
    # 建立传输层 (SyncTcpTransport)
    transport = SyncTcpTransport(host="127.0.0.1")
    # 建立客户端 (SyncModbusClient)
    client = SyncModbusClient(transport)

    data = client.read_holding_registers(0, 10)

except (ConnectError, TimeOutError) as e:
    print(f"Error: {e}")
```

### 示例 2：日志与异常捕获 (Logging & Exceptions)

```python
# ================= OLD =================
from modbuslink import get_logger, CRCError

logger = get_logger("MyApp")

try:
    # ... operation ...
    pass
except CRCError:
    logger.error("CRC校验失败", "CRC check failed")

# ================= NEW =================
# 1. 路径变更: utils -> common
# 2. 异常名变更: CRCError -> CrcError
from modbuslink import get_logger, CrcError

logger = get_logger("MyApp")

try:
    # ... operation ...
    pass
except CrcError:
    # 3. 双语日志支持 (可选)
    logger.error("CRC校验失败", "CRC check failed")
```
