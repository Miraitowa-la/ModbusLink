# ModbusLink v1.4.2 版本发布说明

## 核心更新：TCP 传输层自动恢复与 flush() 机制

### 问题背景

在使用 ModbusLink 处理偶尔响应缓慢（导致超时）的设备时，后续请求常常会收到"事务 ID 不匹配"错误`InvalidReplyError` (`1.4.0` 版本前为 `InvalidResponseError`)。这是因为：

1. **请求 A (TID=100)** 超时
2. 设备延迟响应，TID=100 的响应仍在 TCP 缓冲区中
3. **请求 B (TID=101)** 已发送
4. 请求 A 的延迟响应以 TID=100 到达
5. 由于 TID=100 ≠ 期望的 TID=101，抛出 `InvalidReplyError`
6. 这种连锁反应持续下去，每一次"迟到"的回复都会影响下一个请求

### 解决方案

`1.4.2` 引入了双层防护机制

1. **自动过滤（Auto-Drain）**：在 `send_and_receive` 内部实现了智能过滤。当检测到 TID 对应的响应是陈旧数据时，传输层会自动读取并丢弃该包，继续等待当前事务的正确响应。
2. **手动清空（Manual Flush）**：为 `SyncTcpTransport` 和 `AsyncTcpTransport` 类添加了 `flush()` 方法，用于清空接收缓冲区中的陈旧数据。

### 影响场景

- **Home Assistant 集成**：彻底解决因偶尔超时导致的“不可用”状态，无需频繁重启连接。
- **高延迟网络**：增强了 DTU 或跨网段通信的健壮性。

## API 新增内容

### 1. 同步版本

```python
def flush(self) -> int:
    """
    清空接收缓冲区中的所有待处理数据
    
    Returns:
        丢弃的字节数
    """
```

### 2. 异步版本

```python
async def flush(self) -> int:
    """
    清空接收缓冲区中的所有待处理数据
    
    Returns:
        丢弃的字节数
    """
```

## 使用示例

### 基本用法 - 同步客户端

```python
from modbuslink import SyncModbusClient, SyncTcpTransport, TimeOutError

transport = SyncTcpTransport(host="192.168.1.100", port=502, timeout=2.0)
client = SyncModbusClient(transport)

with client:
    try:
        result = client.read_holding_registers(slave_id=1, start_address=0, quantity=10)
    except TimeOutError:
        # 超时后清空缓冲区
        discarded = transport.flush()
        print(f"Flushed {discarded} bytes")

        # 重试
        result = client.read_holding_registers(slave_id=1, start_address=0, quantity=10)
```

### 基本用法 - 异步客户端

```python
from modbuslink import AsyncModbusClient, AsyncTcpTransport, TimeOutError

transport = AsyncTcpTransport(host="192.168.1.100", port=502, timeout=2.0)
client = AsyncModbusClient(transport)

async with client:
    try:
        result = await client.read_holding_registers(slave_id=1, start_address=0, quantity=10)
    except TimeOutError:
        # 超时后清空缓冲区
        discarded = await transport.flush()
        print(f"Flushed {discarded} bytes")

        # 重试
        result = await client.read_holding_registers(slave_id=1, start_address=0, quantity=10)
```

### Home Assistant 集成模式

```python
import asyncio
from modbuslink import AsyncModbusClient, AsyncTcpTransport, TimeOutError, InvalidReplyError


async def poll_device():
    transport = AsyncTcpTransport(host="192.168.1.100", port=502, timeout=3.0)
    client = AsyncModbusClient(transport)

    poll_interval = 60  # 60 秒轮询一次
    max_consecutive_errors = 3
    consecutive_errors = 0

    await transport.open()

    while True:
        try:
            # 读取设备数据
            registers = await client.read_holding_registers(
                slave_id=1,
                start_address=0,
                quantity=10
            )

            print(f"Data: {registers}")
            consecutive_errors = 0  # 重置错误计数

        except TimeOutError:
            consecutive_errors += 1
            print(f"Timeout (consecutive errors: {consecutive_errors})")

            # 清空缓冲区
            await transport.flush()

            if consecutive_errors >= max_consecutive_errors:
                # 重新连接
                await transport.close()
                await asyncio.sleep(1)
                await transport.open()
                consecutive_errors = 0

        except InvalidReplyError:
            # 事务ID不匹配，清空缓冲区
            await transport.flush()

        await asyncio.sleep(poll_interval)
```

## 实现特点

1. **非阻塞读取**：`flush()` 设计为快速消耗缓冲区，不会造成线程阻塞
2. **安全性**：在未连接状态下调用返回 0，不会抛出异常
3. **自我修复**：传输层现在能容忍一定程度的脏数据，而不会直接抛出异常中断流程。
4. **日志记录**：当丢弃数据时会记录警告日志
5. **类似 libmodbus**：API 设计参考了 libmodbus 的 `modbus_flush()` 函数
