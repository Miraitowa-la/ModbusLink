# ModbusLink v1.3.4 版本发布说明

## 新功能：异步传输层并发安全修复

### 并发操作问题修复

解决了在异步客户端中进行并发操作时出现的连接冲突问题。在多任务并发访问同一连接时，会出现 `readexactly() called while another coroutine is already waiting for incoming data` 错误。

### 核心修复内容

为所有异步传输层添加了并发控制锁，确保多协程环境下的线程安全：

### 修复的传输层类

- AsyncTcpTransport - 异步TCP传输层
- AsyncRtuTransport - 异步RTU传输层
- AsyncAsciiTransport - 异步ASCII传输层

### 修复方案

```Python
# 使用锁确保并发安全
async with self._lock:
    # 完整的发送-接收流程
    response_pdu = await self._receive_exact(pdu_length)
    return response_pdu
```

## 文件修改内容

- src\modbuslink\transport\async_tcp.py：添加并发控制锁
- src\modbuslink\transport\async_rtu.py：添加并发控制锁
- src\modbuslink\transport\async_ascii.py：添加并发控制锁

## 解决议题（ #4 ）

- 修复了异步客户端在并发操作时的连接冲突问题
- 确保异步传输层在多协程环境下的线程安全
- 提高了异步客户端在高并发场景下的稳定性
